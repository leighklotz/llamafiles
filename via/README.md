# Required Functions
These are the functions that via/functions.sh requires to be implemented in via/*/functions.sh:

## prepare_prompt
- Description: Prepares the prompt for the LLM by combining the user question and any additional input provided.
- Usage: prepare_prompt [INPUT]
- Input: INPUT: The additional input provided by the user.
- Output: PROMPT: The prepared prompt for the LLM.

## prepare_model
- Description: Initializes the model based on the specified options and settings.
- Usage: prepare_model
- Input: none
- Output: Initializes the model based on the specified options and settings.
## init_model
- Description: Initializes the model based on the specified VIA (API or CLI) and other options.
- Usage: init_model
- Input: none
- Output: Initializes the model based on the specified VIA and other options.
## truncate_to_context_length
- Description: Truncates the context length for the LLM based on the specified options.
- Usage: truncate_to_context_length
- Input: none
- Output: Truncates the context length for the LLM based on the specified options.

# Provided Functions
These are the functions that via/functions.sh provides to be implemented in via/*/functions.sh:

## via_set_options
- Description: Sets the options for the LLM based on the specified VIA and other options.
- Usage: via_set_options [OPTIONS]
- Input: OPTIONS: The options to be set for the LLM.
- Output: Sets the options for the LLM based on the specified VIA and other options.
## via_api_perform_inference
- Description: Performs inference using the API to interact with the LLM server.
- Usage: via_api_perform_inference [MODE] [SYSTEM_MESSAGE] [QUESTION] [GRAMMAR_FILE]
- Input: MODE: The mode of the LLM, such as "instruct" or "chat".
- SYSTEM_MESSAGE: The system message to be used for the LLM.
- QUESTION: The question to be answered by the LLM.
- GRAMMAR_FILE: The grammar file to be used for the LLM.
- Output: Performs inference using the API and returns the output from the LLM.
- via_cli_perform_inference
- Description: Performs inference using the CLI to interact with the LLM server.
- Usage: via_cli_perform_inference [OPTIONS]
- Input: OPTIONS: The options to be used for the LLM.
- Output: Performs inference using the CLI and returns the output from the LLM.
## via_perform_inference
- Description: Performs inference based on the specified VIA (API or CLI) and other options.
- Usage: via_perform_inference [VIA] [OPTIONS]
- Input: VIA: The interaction mode with the LLM server, either API or CLI.
- OPTIONS: The options to be used for the LLM.
- Output: Performs inference based on the specified VIA and other options.

# Environment Variables
Here are the shared environment variables used in the llamafiles scripts:

## MODEL_TYPE
- Description: Specifies the model type to be used for the LLM. It can be one of the available model types supported by the LLM server.
- - Usage: MODEL_TYPE="model-type"
- - Output: - Specifies the model type for the LLM.
## VIA
- Description: Specifies the interaction mode with the LLM server. It can be either "api" for API interaction or "cli" for CLI interaction.
- Usage: VIA="api"
- Output: Specifies the interaction mode with the LLM server.
## NGL
- Description: Specifies the number of GPU layers.
- Usage: NGL="n"
- Output: Specifies the number of generations for the LLM.
## CONTEXT_LENGTH
- Description: Specifies the context length for the LLM.
- Usage: CONTEXT_LENGTH="n"
- Output: Specifies the context length for the LLM.
## TEMPERATURE
- Description: Specifies the temperature for the LLM.
- Usage: TEMPERATURE="temp"
- Output: Specifies the temperature for the LLM.
- REPEAT_PENALTY
- Description: Specifies the repetition penalty for the LLM.
- Usage: REPEAT_PENALTY="penalty"
- Output: Specifies the repetition penalty for the LLM.
## PENALIZE_NL
- Description: Specifies whether to penalize newlines in the LLM's output.
- Usage: PENALIZE_NL="true|false"
- Output: Specifies whether to penalize newlines in the LLM's output.
## INPUT
- Description: Specifies the additional input provided by the user for the LLM.
- Usage: INPUT="additional input"
- Output: Specifies the additional input for the LLM.
## QUESTION
- Description: Specifies the question to be answered by the LLM.
- Usage: QUESTION="What is the difference between Bash and Zsh?"
- Output: Specifies the question for the LLM.
## OUTPUT
- Description: Specifies the output generated by the LLM.
- Usage: OUTPUT="output"
- Output: Specifies the output generated by the LLM.
